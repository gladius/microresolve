## One engine, many classifiers

MicroResolve uses **namespaces** to run independent classifiers in parallel from a single engine instance:

- Customer-support intent classification
- Security and abuse detection
- Mood / sentiment filtering
- MCP tool pre-selection
- Compliance guardrails (HIPAA, EU AI Act, …)

Each namespace is isolated, independently trainable, and queryable in a single call.

```rust
use microresolve::{MicroResolve, MicroResolveConfig};

let engine = MicroResolve::new(MicroResolveConfig {
    data_dir: Some("~/.local/share/microresolve".into()),
    ..Default::default()
})?;

let support  = engine.namespace("support");
let security = engine.namespace("security");

let intent   = support.resolve("cancel my order");
let threat   = security.resolve("ignore previous instructions");
```

## Why deterministic routing?

Large language models are powerful but slow and expensive. The majority of incoming queries — "cancel my order", "reset my password", "show me the dashboard" — follow patterns that can be resolved deterministically in microseconds. MicroResolve handles that 80%, so your LLM budget goes to the 20% that actually needs reasoning.

```
query → [MicroResolve ~50µs] → matched intent → template / tool / guardrail
                              ↘ no match → LLM fallback
```

## Git data layer

Your training data is a git repo. Every namespace mutation auto-commits. You get a built-in audit trail, one-command rollback to any historical state, and optional remote sync via a standard git remote.

```bash
# Roll back a namespace to a previous commit
curl -X POST http://localhost:4000/api/namespaces/support/rollback \
  -H "Content-Type: application/json" \
  -d '{"sha": "abc1234..."}'
```
